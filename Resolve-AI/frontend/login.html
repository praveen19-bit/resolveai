How It Works

From Complaint to Resolution

A fully automated 6-step journey — powered by AI, RAG, and WhatsApp.

01
User Submits Complaint
Student or Faculty types the issue in the AI chat interface. They can upload a proof photo and pin their exact campus location on the map.
Chat Interface + Photo + GPS Pin
02
AI Agent Classifies
LangChain agent with Groq Llama analyzes the text (and image if provided). It assigns category, priority, department, and generates a suggested action — in under 2 seconds.
Groq Llama + Vision Model
03
Duplicate Check + RAG Lookup
FAISS searches for similar open tickets — if a duplicate is found, the new complaint is linked instead of creating a new ticket. RAG also retrieves the exact contact person for the department.
FAISS + MiniLM Embeddings
04
WhatsApp Alert Sent
n8n automation fires a WhatsApp message to the exact right technician with ticket details, location, priority, and AI-suggested action. SLA timer starts automatically.
n8n + WhatsApp Business API
05
Maintenance Team Resolves
Technician fixes the issue and uploads proof photos via the maintenance dashboard. Status updates to Resolved and is stored in MongoDB with timestamp.
Maintenance Dashboard + Cloudinary
06
User Verifies & Closes
User receives a WhatsApp notification with proof images. They verify on the portal and rate the resolution 1–5 stars. Analytics dashboard updates in real-time.
Star Rating + Real-time Analytics
